Overview

Learn what Docs is all about and how you can use it.

What does it do?

Docs is a simple HTML and CSS template for documentation of a project. It is made to provide documentation for smaller projects that don't need a whole user guide. An ideal use of Docs would be for a project with 5-6 basic sections that need to be covered. There is no limit to how many pages you can create. However, anything excessive would make better use of a full feature documentation tool.

What are some good uses for Docs?

Docs is perfect for your JavaScript plugin, basic framework, module, or small API.

How hard is it to implement?

There's only a few main steps to take to get your rotator up and running.

Copy and rename the _template folder in your project:

  1. Copy the _template folder from the root of the repo
  2. Paste it into your project and rename it to docs or whatever you'd like

Replace the following variables in the index.html file:

  • _Project_ — becomes your project name; (E.g. Docs or Docs API)
  • _Project Description Short_ — becomes your project short description (one sentence); (E.g. A basic HTML/CSS template for code documentation.; used in page title)
  • _Project Description_ — becomes your project description; (E.g. A basic HTML/CSS template built to help you quickly set up documentation for your project)
  • _Company_ — becomes the company or person who holds the license for this project; (E.g. cmfolio)
  • _Company URL_ — becomes your company or personal url; (E.g. http://web.cmfolio.com)

How can I customize Docs?

Since you already have full control over the HTML you generate, we provide some basic themes for you to start with. You can either take out themes and modify them or create a new one from scratch.

What are the requirements?

Docs is purely HTML, CSS and JavaScript. You do not need any specific backend language or database to run Docs. This is done to ensure it can run on a machine locally so you can include documentation with your project regardless of what platform it is on.
